DWG-01PROBLEM STATEMENT

لماذا لا تكفينا الـ Array؟

قبل أن نتعلم Collections، يجب أن نفهم المشكلة التي جاءت لتحلّها.

في بداية تعلّمك للبرمجة، تعرّفت على Array كوسيلة لتخزين عدة قيم من نفس النوع. لكن الـ Array له عيب جوهري: حجمه ثابت منذ لحظة إنشائه. إن أنشأت مصفوفة لتخزين 5 طلاب، لن تستطيع لاحقاً إضافة طالب سادس دون إعادة إنشاء المصفوفة بالكامل ونسخ كل العناصر القديمة إليها يدوياً.

Program.cs — المشكلة مع Array
string[] students = new string[3];
students[0] = "أحمد";
students[1] = "سارة";
students[2] = "محمد";

// students[3] = "ليلى"; // ❌ خطأ! IndexOutOfRangeException
// الحجم 3 ثابت، لا يمكن إضافة عنصر رابع مباشرة
المشكلة بالأرقام لإضافة عنصر واحد لمصفوفة ممتلئة، يجب: إنشاء مصفوفة جديدة بحجم أكبر، نسخ كل العناصر القديمة (نسخ O(n))، ثم إضافة العنصر الجديد. هذا مكلف جداً إن تكرر العملية مئات المرات.

هنا يأتي دور Collections: مجموعة من الكلاسات الجاهزة في مكتبة #C تتولى عنك إدارة الذاكرة والتكبير والتصغير تلقائياً، وتمنحك أدوات جاهزة (مثل الترتيب والبحث والحذف) بدل أن تكتبها بنفسك في كل مرة.

تعريف أكاديمي الـ Collection هي بنية بيانات (Data Structure) توفر طريقة منظّمة لتخزين، الوصول إلى، وإدارة مجموعة من العناصر المرتبطة ببعضها منطقياً، مع عمليات جاهزة مثل الإضافة (Add)، الحذف (Remove)، البحث (Search)، والتكرار (Iteration).
DWG-02CLASSIFICATION MAP

خريطة تصنيف الـ Collections

قبل الدخول في التفاصيل، لنرسم الصورة الكبيرة لتعرف أين يقع كل نوع.

أولاً: عائلتان مختلفتان

في #C يوجد مكتبتان للـ Collections، ويجب أن تتعلّم استخدام الثانية فقط في المشاريع الحديثة:

المكتبة (namespace)الوصفهل ننصح باستخدامها؟
System.Collectionsقديمة (Non-Generic)، تخزن كل شيء كـ object، تحتاج Boxing/Unboxing وCasting يدوي، وغير آمنة من ناحية الأنواع.تجنّبها قدر الإمكان
System.Collections.Genericحديثة (Generic)، تستخدم <T> لتحديد نوع العناصر بدقة، أسرع، وآمنة وقت الترجمة (Compile-time).استخدمها دائماً
الفرق بين Generic و Non-Generic
// ❌ الطريقة القديمة (ArrayList) — غير آمنة
ArrayList oldList = new ArrayList();
oldList.Add(10);
oldList.Add("نص"); // يسمح بخلط الأنواع! خطأ شائع يكتشف وقت التشغيل فقط
int x = (int)oldList[0]; // يحتاج Casting يدوي

// ✅ الطريقة الحديثة (List<T>) — آمنة
List<int> newList = new List<int>();
newList.Add(10);
// newList.Add("نص"); // ❌ خطأ يكتشفه المترجم فوراً (Compile-time error)
int y = newList[0]; // بدون Casting

ثانياً: التسلسل الهرمي للواجهات (Interfaces)

كل أنواع الـ Collections في #C تبني فوق واجهات أساسية مشتركة. فهم هذا الهرم يفسّر لماذا تستطيع استخدام foreach مع جميع الأنواع، رغم اختلاف طريقة عملها الداخلية.

* Stack و Queue لا يطبّقان IList مباشرة لأنهما يقيّدان الوصول العشوائي عمداً — سنوضح السبب لاحقاً
لماذا هذا مهم؟ لأن foreach يعمل مع أي نوع يطبّق IEnumerable<T> فقط — وكل الـ Collections تطبّقه في القمة. هذا يعني أنك تتعلّم foreach مرة واحدة وتستخدمه مع كل الأنواع القادمة.
DWG-03DYNAMIC ARRAY

List<T> — المصفوفة الديناميكية

الأكثر استخداماً بين كل الـ Collections. فكّر بها كمصفوفة تكبر وتصغر تلقائياً.

List<T> تخزّن عناصر مرتّبة بترتيب الإدخال (Ordered)، وتسمح بالوصول المباشر عبر الفهرس (Index) تماماً مثل المصفوفة، لكنها تكبر تلقائياً عند الحاجة. هي الخيار الافتراضي عندما تحتاج فقط "قائمة عناصر" دون متطلبات خاصة.

أهم الخصائص والدوال

العضوالوظيفة
Add(item)إضافة عنصر في نهاية القائمة
Insert(i, item)إدراج عنصر في موضع محدد
Remove(item)حذف أول ظهور لقيمة معيّنة
RemoveAt(i)حذف عنصر حسب موضعه
Contains(item)هل القيمة موجودة؟ (true/false)
IndexOf(item)موضع أول ظهور للقيمة، أو -1
Sort()فرز العناصر تصاعدياً
Countعدد العناصر الحالي
الفهرسة تبدأ دائماً من الصفر — وهذا مصدر شائع للأخطاء عند المبتدئين

مثال كامل وعملي

سيناريو: إدارة قائمة بدرجات طلاب في مادة دراسية، مع حساب المعدل وعرض الأعلى درجة.

Program.cs
using System;
using System.Collections.Generic;
using System.Linq;

List<int> grades = new List<int>();
grades.Add(85);
grades.Add(92);
grades.Add(78);
grades.Add(95);

Console.WriteLine($"عدد الدرجات: {grades.Count}");

grades.Sort(); // فرز تصاعدي
Console.WriteLine($"أعلى درجة: {grades[grades.Count - 1]}");
Console.WriteLine($"أقل درجة: {grades[0]}");

double average = grades.Average(); // من LINQ
Console.WriteLine($"المعدل: {average:F2}");

grades.Remove(78); // حذف قيمة محددة
Console.WriteLine($"بعد الحذف، العدد: {grades.Count}");

foreach (int g in grades)
{
    Console.WriteLine($"- {g}");
}
>_ OUTPUT
عدد الدرجات: 4 أعلى درجة: 95 أقل درجة: 78 المعدل: 87.50 بعد الحذف، العدد: 3 - 85 - 92 - 95

DWG-04KEY → VALUE LOOKUP

Dictionary<TKey, TValue> — دفتر العناوين

كل عنصر مكوّن من زوج: مفتاح فريد، وقيمة مرتبطة به. تماماً مثل دفتر هاتف.

بدل البحث عن عنصر عبر فهرس رقمي (0، 1، 2...)، تبحث في Dictionary عبر مفتاح (Key) من أي نوع تختاره (نص، رقم، حتى كائن). كل مفتاح يجب أن يكون فريداً (Unique) — لا يمكن تكراره. البحث بالمفتاح سريع جداً تقريباً O(1) بفضل بنية داخلية تسمى Hash Table.

أهم الخصائص والدوال

العضوالوظيفة
dict[key] = valueإضافة أو تحديث قيمة لمفتاح
Add(key, value)إضافة (يرمي خطأ إن كان المفتاح مكرراً)
ContainsKey(key)هل المفتاح موجود؟
TryGetValue(key, out v)قراءة آمنة بدون خطر استثناء
Remove(key)حذف زوج كامل عبر مفتاحه
Keys / Valuesالحصول على كل المفاتيح أو كل القيم
لا يوجد ترتيب مضمون للعناصر — لا تعتمد على ترتيب الإدخال

مثال كامل وعملي

سيناريو: عدّاد لتكرار الكلمات في جملة — تطبيق كلاسيكي يُظهر قوة Dictionary.

Program.cs
string sentence = "الكتاب على الطاولة والكتاب جديد";
string[] words = sentence.Split(' ');

Dictionary<string, int> wordCount = new Dictionary<string, int>();

foreach (string word in words)
{
    if (wordCount.ContainsKey(word))
    {
        wordCount[word]++; // الكلمة موجودة، زِد العداد
    }
    else
    {
        wordCount[word] = 1; // كلمة جديدة، ابدأ بـ 1
    }
}

foreach (KeyValuePair<string, int> pair in wordCount)
{
    Console.WriteLine($"{pair.Key} : {pair.Value} مرة");
}

// قراءة آمنة بدون استثناء
if (wordCount.TryGetValue("جديد", out int count))
{
    Console.WriteLine($"كلمة 'جديد' تكررت {count} مرة");
}
>_ OUTPUT
الكتاب : 2 مرة على : 1 مرة الطاولة : 1 مرة والكتاب : 1 مرة جديد : 1 مرة كلمة 'جديد' تكررت 1 مرة
تحذير شائع استخدام dict[key] للقراءة عندما يكون المفتاح غير موجود يرمي KeyNotFoundException. استخدم دائماً TryGetValue أو ContainsKey أولاً عند عدم التأكد من وجود المفتاح.

DWG-05FIFO — FIRST IN, FIRST OUT

Queue<T> — الطابور

بالضبط مثل طابور البنك: أول من يدخل، هو أول من يُخدم ويخرج.

الـ Queue<T> تطبّق مبدأ FIFO (First In, First Out). لا يمكنك الوصول لعنصر عشوائي بفهرس مثل List — يمكنك فقط التعامل مع طرفي الطابور: الإضافة من الخلف، والإخراج من الأمام. تُستخدم في معالجة المهام بالترتيب، أنظمة الطباعة، وإدارة الطلبات.

أهم الدوال

العضوالوظيفة
Enqueue(item)إضافة عنصر في نهاية الطابور
Dequeue()إخراج وإرجاع العنصر الأول (يحذفه)
Peek()معاينة العنصر الأول دون حذفه
Countعدد العناصر الحالي في الطابور
مبدأ FIFO: من يصل أولاً يُخدم أولاً

مثال كامل وعملي

سيناريو: نظام طابور تذاكر بسيط في عيادة طبية.

Program.cs
Queue<string> clinicQueue = new Queue<string>();

clinicQueue.Enqueue("مريض: خالد");
clinicQueue.Enqueue("مريض: منى");
clinicQueue.Enqueue("مريض: يوسف");

Console.WriteLine($"عدد المنتظرين: {clinicQueue.Count}");
Console.WriteLine($"التالي في الدور: {clinicQueue.Peek()}"); // معاينة فقط

while (clinicQueue.Count > 0)
{
    string current = clinicQueue.Dequeue(); // إخراج وخدمة
    Console.WriteLine($"جاري الكشف على: {current}");
}

Console.WriteLine($"العيادة فارغة الآن: {clinicQueue.Count == 0}");
>_ OUTPUT
عدد المنتظرين: 3 التالي في الدور: مريض: خالد جاري الكشف على: مريض: خالد جاري الكشف على: مريض: منى جاري الكشف على: مريض: يوسف العيادة فارغة الآن: True

DWG-06LIFO — LAST IN, FIRST OUT

Stack<T> — المكدّس

مثل كومة أطباق: آخر طبق وضعته هو أول طبق سترفعه.

الـ Stack<T> تطبّق مبدأ LIFO (Last In, First Out): آخر عنصر يُضاف هو أول عنصر يُحذف. تُستخدم في ميزة "تراجع" (Undo) بالبرامج، تتبّع استدعاءات الدوال (Call Stack)، والتحقق من توازن الأقواس في الأكواد البرمجية.

أهم الدوال

العضوالوظيفة
Push(item)إضافة عنصر فوق المكدّس
Pop()إخراج وإرجاع العنصر العلوي (يحذفه)
Peek()معاينة العنصر العلوي دون حذفه
Countعدد العناصر الحالي في المكدّس
مبدأ LIFO: آخر عنصر دخل هو أول عنصر يخرج

مثال كامل وعملي

سيناريو: التحقق من توازن الأقواس في نص برمجي — استخدام كلاسيكي للمكدّس.

Program.cs
bool IsBalanced(string code)
{
    Stack<char> stack = new Stack<char>();

    foreach (char c in code)
    {
        if (c == '(') stack.Push(c);
        else if (c == ')')
        {
            if (stack.Count == 0) return false; // إغلاق بدون فتح
            stack.Pop();
        }
    }
    return stack.Count == 0; // متوازن إن لم يتبقَّ شيء
}

Console.WriteLine(IsBalanced("(a + (b * c))")); // True
Console.WriteLine(IsBalanced("(a + b))"));      // False
>_ OUTPUT
True False

اكتب دالة #C باسم ReverseWord تستقبل نصاً، وتستخدم Stack<char> لعكس ترتيب حروفه (بدفع كل حرف ثم سحبه)، ثم اطبع النتيجة لكلمة "برمجة".

✅ الحل المقترح
Solution.cs
string ReverseWord(string word)
{
    Stack<char> letters = new Stack<char>();

    foreach (char c in word)
        letters.Push(c);

    string reversed = "";
    while (letters.Count > 0)
        reversed += letters.Pop();

    return reversed;
}

Console.WriteLine(ReverseWord("برمجة")); // ةجمرب
DWG-07UNIQUE VALUES ONLY

HashSet<T> — مجموعة بلا تكرار

إن حاولت إضافة قيمة موجودة بالفعل، يتجاهلها بصمت دون خطأ.

الـ HashSet<T> تخزّن قيماً فريدة فقط (بلا تكرار)، ولا تضمن أي ترتيب معيّن للعناصر. مثاليّة عندما يهمك فقط "هل هذه القيمة موجودة؟" بسرعة كبيرة، أو عند الحاجة لعمليات المجموعات الرياضية (اتحاد، تقاطع، فرق) كما في الرياضيات.

أهم الدوال

العضوالوظيفة
Add(item)إضافة قيمة (ترجع false إن كانت مكررة)
Remove(item)حذف قيمة معيّنة
Contains(item)فحص الوجود بسرعة كبيرة O(1)
UnionWith(other)اتحاد مع مجموعة أخرى
IntersectWith(other)تقاطع (العناصر المشتركة فقط)
ExceptWith(other)الفرق (احذف عناصر موجودة بمجموعة أخرى)
المجموعة تتجاهل القيم المكررة تلقائياً دون رمي استثناء

مثال كامل وعملي

سيناريو: إزالة التكرار من قائمة بريد إلكتروني، ثم استخراج العملاء المشتركين بين حملتين تسويقيتين.

Program.cs
List<string> rawEmails = new List<string>
{
    "a@x.com", "b@x.com", "a@x.com", "c@x.com", "b@x.com"
};

HashSet<string> uniqueEmails = new HashSet<string>(rawEmails);
Console.WriteLine($"عدد البريد الأصلي: {rawEmails.Count}");      // 5
Console.WriteLine($"بعد إزالة التكرار: {uniqueEmails.Count}");   // 3

HashSet<string> campaignA = new HashSet<string> { "a@x.com", "b@x.com" };
HashSet<string> campaignB = new HashSet<string> { "b@x.com", "c@x.com" };

campaignA.IntersectWith(campaignB); // الآن campaignA تحتوي فقط المشترك
Console.WriteLine("العملاء المشتركون بين الحملتين:");
foreach (string email in campaignA)
    Console.WriteLine($"- {email}");

DWG-08SPECIALIZED STRUCTURES

أنواع إضافية تستحق المعرفة

أقل استخداماً يومياً، لكن ضرورية لفهم الصورة الكاملة.

LinkedList<T> — القائمة المترابطة

على عكس List<T> التي تخزّن عناصرها متجاورة في الذاكرة، تخزّن LinkedList<T> كل عنصر في "عقدة" (Node) منفصلة، وكل عقدة تحمل إشارة (Pointer) للعقدة التالية والسابقة. ميزتها: الإضافة والحذف من المنتصف سريعة جداً O(1) لأنها لا تحتاج إزاحة بقية العناصر. عيبها: لا يوجد وصول مباشر بفهرس، يجب المرور عبر العقد بالترتيب.

كل عقدة تشير إلى التالية (وفي LinkedList ثنائية الاتجاه، إلى السابقة أيضاً)
Program.cs
LinkedList<string> playlist = new LinkedList<string>();
playlist.AddLast("أغنية 1");
playlist.AddLast("أغنية 2");
playlist.AddFirst("مقدمة"); // إضافة سريعة في البداية

foreach (string track in playlist)
    Console.WriteLine(track);
// الناتج: مقدمة - أغنية 1 - أغنية 2

SortedList<TKey,TValue> و SortedDictionary<TKey,TValue>

نسخة من Dictionary لكنها تحافظ على ترتيب العناصر تلقائياً حسب المفتاح في كل مرة تتصفّحها. SortedList أسرع في القراءة بالفهرس وأقل استهلاكاً للذاكرة، بينما SortedDictionary أسرع في الإضافة والحذف المتكرر.

Program.cs
SortedDictionary<string, int> scores = new SortedDictionary<string, int>();
scores["زياد"] = 70;
scores["أحمد"] = 90;
scores["مريم"] = 85;

foreach (var pair in scores) // تُطبع مرتّبة أبجدياً تلقائياً حسب المفتاح
    Console.WriteLine($"{pair.Key}: {pair.Value}");
// الناتج: أحمد: 90 - زياد: 70 - مريم: 85
متى تستخدم أيهما؟ استخدم LinkedList فقط عند إضافة/حذف متكرر جداً من المنتصف. استخدم SortedDictionary عند الحاجة لترتيب دائم بالمفتاح. في باقي الحالات، List و Dictionary العاديان كافيان وأبسط.

تمرين 6

اكتب نظاماً بسيطاً لإدارة طلاب: استخدم List<string> لتخزين أسماء الطلاب، و Dictionary<string, int> لتخزين درجة كل طالب. اطبع اسم الطالب صاحب أعلى درجة دون استخدام Sort، بالاعتماد على المرور على القائمة ومقارنة القيم في القاموس.

✅ الحل المقترح
Solution.cs
List<string> names = new List<string> { "هبة", "كريم", "ريم", "سامي" };
Dictionary<string, int> grades = new Dictionary<string, int>
{
    { "هبة", 88 }, { "كريم", 95 }, { "ريم", 79 }, { "سامي", 91 }
};

string topStudent = names[0];
int topGrade = grades[topStudent];

foreach (string name in names)
{
    if (grades[name] > topGrade)
    {
        topGrade = grades[name];
        topStudent = name;
    }
}

Console.WriteLine($"الأعلى تحصيلاً: {topStudent} بدرجة {topGrade}");
// الناتج: الأعلى تحصيلاً: كريم بدرجة 95
تمرين 7

اكتب دالة IsPalindrome(string text) تتحقق هل النص يُقرأ نفسه من الاتجاهين (مثل "اتاتا" أو "level")، باستخدام Queue<char> و Stack<char> معاً: ضع كل الحروف في الاثنين، ثم قارن ما يخرج من Dequeue مقابل Pop حرفاً حرفاً.

✅ الحل المقترح
Solution.cs
bool IsPalindrome(string text)
{
    Queue<char> fromStart = new Queue<char>();
    Stack<char> fromEnd = new Stack<char>();

    foreach (char c in text)
    {
        fromStart.Enqueue(c); // يخرج من البداية (Dequeue)
        fromEnd.Push(c);      // يخرج من النهاية (Pop)
    }

    while (fromStart.Count > 0)
    {
        if (fromStart.Dequeue() != fromEnd.Pop())
            return false; // اختلاف يعني أنه ليس Palindrome
    }
    return true;
}

Console.WriteLine(IsPalindrome("level"));  // True
Console.WriteLine(IsPalindrome("سيارة"));  // False
تمرين 8 — تحدّي شامل

لديك قائمتان من اهتمامات مستخدمَين على منصة تواصل: user1Interests و user2Interests (كل منهما List<string> وقد تحتوي تكراراً داخلياً). اكتب كوداً يستخدم HashSet<T> ليطبع: الاهتمامات المشتركة بينهما (Intersect)، ثم عدد الاهتمامات الفريدة الكلية بين الاثنين مجتمعين (Union).

✅ الحل المقترح
Solution.cs
List<string> user1Interests = new List<string> { "برمجة", "قراءة", "برمجة", "رياضة" };
List<string> user2Interests = new List<string> { "طبخ", "رياضة", "قراءة" };

HashSet<string> set1 = new HashSet<string>(user1Interests);
HashSet<string> set2 = new HashSet<string>(user2Interests);

HashSet<string> common = new HashSet<string>(set1);
common.IntersectWith(set2);

Console.WriteLine("اهتمامات مشتركة:");
foreach (string i in common)
    Console.WriteLine($"- {i}");

HashSet<string> allInterests = new HashSet<string>(set1);
allInterests.UnionWith(set2);
Console.WriteLine($"إجمالي الاهتمامات الفريدة: {allInterests.Count}");
// مشتركة: قراءة - رياضة
// الإجمالي الفريد: 5 (برمجة، قراءة، رياضة، طبخ)... تحقّق بنفسك من العدّ!
DWG-11SUMMARY

خلاصة المحاضرة

في النهاية، الاختيار الصحيح للـ Collection هو الفرق بين كود بطيء معقّد، وكود سريع واضح.

تذكّر دوماً السؤال الأهم قبل كتابة أي سطر كود: "ما طبيعة البيانات التي أتعامل معها، وكيف سأصل إليها؟" إن كانت قائمة عامة، فكّر بـ List. إن كانت أزواج مفتاح-قيمة، فكّر بـ Dictionary. إن كان الترتيب يهم (الأول يخدم أولاً، أو الأحدث يُعالج أولاً)، فكّر بـ Queue أو Stack. وإن كان منع التكرار هو الهدف، فـ HashSet هو صديقك.

تكليف إضافي للمراجعة الذاتية ارجع لكل تمرين في هذه الصفحة، وحاول حلّه بنفسك على ورقة قبل الضغط على "إظهار الحل" — هذا أهم خطوة لترسيخ الفهم الحقيقي بدل الحفظ.
